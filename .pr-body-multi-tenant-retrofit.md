# Multi-tenant retrofit + Athena cutover unblock

## Summary

Adds the foundational multi-tenant model (`organizations`, `organization_members`, `organization_id` on every PHI-bearing table) and fixes a pre-existing **critical bug** that would have blocked the Athena pilot doctor from connecting their EHR.

## What changed

### New schema + migrations
- `lib/db/src/schema/organizations.ts` â€” clinic-level org row (name, slug, specialty, NPI, optional address; soft-delete via `deleted_at`).
- `lib/db/src/schema/organization-members.ts` â€” typed role enum: `owner | admin | provider | billing | ma | viewer` (matches the spec's role list). `clinical_read_granted` for billing-role users who need clinical access. `is_active` so historical memberships stay on file for audit attribution.
- `organization_id NOT NULL` added to: `patients`, `notes`, `recording_jobs`, `ehr_connections`, `note_templates`, `provider_phrase_mappings`, `provider_note_defaults`. Nullable on `audit_log` (for system events) and `ehr_oauth_states` (legacy in-flight states; backfilled, new states populate it).
- `patients.mrn` uniqueness changed from globally unique to `UNIQUE (organization_id, mrn)` so two clinics can legitimately share an MRN.
- `sessions.active_organization_id` added so PHI routes have tenant context without a join per request.
- Migration **0021** is hand-edited as a 9-step backfill-safe sequence (seed `org_default`, promote first-founder or oldest-user to owner, add columns nullable, UPDATE, set NOT NULL, FKs, swap `mrn` unique). Uses `gen_random_uuid()` for member ids â€” required because Drizzle's `$defaultFn` is TS-side only and raw SQL inserts otherwise hit NOT NULL on `id`.

### Enforcement
- New helper `lib/active-org.ts:getActiveOrgId(req, res)` â€” returns the active org or 409s with `no_active_organization`. Used by every route that touches PHI.
- `require-auth.ts` threads `req.activeOrganizationId` from the session.
- Every PHI route (patients, notes, recordings, templates, phrase-mappings, note-defaults, ehr-oauth) now scopes every SELECT/UPDATE/DELETE by `organization_id`. Cross-tenant patient/note IDs return **404** (not 403) so existence isn't leaked.
- **Signup wraps user + org + owner membership in a single transaction.** Without this, every brand-new user would hit 409 `no_active_organization` on their first PHI request.
- **Audit middleware writes `organization_id`** from `req.activeOrganizationId` so audit rows are tenant-scoped at write time.

### EHR OAuth tenancy
- `ehr_oauth_states` carries `organization_id`, captured at `/start` time. Locks the tenant intent against a mid-flow org switch.
- `completeOauthFlow` throws if `pending.organizationId` is null (defense for legacy in-flight states).
- `upsertConnection` requires `organizationId`; `ehr_connections` rows are now tenant-scoped.
- `upsertCernerPatientFromLaunch` scopes the MRN dedup by org and writes the new patient row to the launch org.
